¿Qué es un middleware?
  Un middleware es un software que se sitúa entre diferentes aplicaciones o sistemas para facilitar la comunicación y el intercambio de datos. En otras palabras, es una capa intermedia que actúa como puente entre distintos componentes de un sistema, permitiendo que se comuniquen de forma más eficiente y segura.

¿Cuál es el significado del término “MEAN stack” ?
  El término "MEAN stack" es un acrónimo que representa un conjunto de tecnologías de software utilizadas para construir aplicaciones web dinámicas y escalables. MEAN es un acrónimo que se compone de las siglas de MongoDB, Express, Angular y Node.js.

¿Cuáles son las características/funcionalidades que Express ofrece como middleware?
  -Manejo de rutas: Express permite definir rutas para diferentes URLs y métodos HTTP, y asociar controladores o funciones que manejen las solicitudes correspondientes.
  -Gestión de solicitudes y respuestas: Express proporciona métodos y propiedades para trabajar con las solicitudes y respuestas HTTP, como parsear los parámetros de la URL, el cuerpo de la solicitud, configurar las cabeceras de la respuesta, entre otros.
  -Manejo de errores: Express ofrece un mecanismo para manejar errores de forma centralizada, lo que permite evitar que la aplicación se bloquee o muestre errores no controlados.
  -Middleware personalizado: Express permite definir middleware personalizado que se ejecuta en cada solicitud, lo que permite agregar funcionalidades adicionales a la aplicación, como la autenticación, la compresión de datos, el manejo de cookies, entre otros.
  -Motor de plantillas: Express incluye un motor de plantillas que permite generar vistas HTML dinámicas utilizando datos dinámicos.
  -Integración con otros middleware y herramientas: Express es compatible con una amplia variedad de middleware y herramientas, como Passport para la autenticación, Morgan para el registro de solicitudes HTTP, Helmet para la seguridad de la aplicación, entre otros.

¿Qué es una ruta en express?
  En Express, una ruta se refiere a una combinación de una URL y un método HTTP que se utiliza para manejar solicitudes entrantes en una aplicación web. Las rutas son definidas en la aplicación utilizando el objeto app proporcionado por Express.

¿Qué es un “route handler” en express y para qué se utiliza?
  Un "route handler" en Express se refiere a una función que se utiliza para manejar solicitudes HTTP en una ruta específica. En otras palabras, el route handler es la función que se ejecuta cuando se realiza una solicitud a una ruta específica en una aplicación Express.

¿Cuáles son los 2 parámetros necesarios para crear una ruta para el método GET con express, ejemplo: app.get()?
  Para crear una ruta para el método GET con Express utilizando el método app.get(), es necesario especificar dos parámetros: la ruta (path) y el controlador (handler).

¿Qué es lo que hace la instrucción res.send?
  En Express, la instrucción res.send() se utiliza para enviar una respuesta HTTP al cliente que realiza la solicitud. La función res.send() toma un parámetro, que puede ser un objeto, una cadena, un array, o cualquier otro tipo de datos que se pueda convertir en una respuesta HTTP.
